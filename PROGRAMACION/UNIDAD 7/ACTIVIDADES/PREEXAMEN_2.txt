Ejercicio 1 - Biblioteca (ArrayList Obligatorio)
Desarrollar una aplicación que gestione los libros de una biblioteca pública.

Requisitos obligatorios
Crear una clase llamada Libro.
Utilizar como estructura principal: ArrayList<Libro>.

Datos de cada libro

título (String)

isbn (String) → único, no se permite duplicado

paginas (int)

autor (String)

prestado (boolean)

Clase Libro
Debe incluir:

Constructor/es necesarios.

Getters y setters.

Método toString() que muestre todos los datos.

Condiciones iniciales
Crear 5 libros en tiempo de diseño.
Todos estarán inicialmente con prestado = false.

Opciones del menú

Alta de libro: Registrar un nuevo libro. No se permiten ISBN duplicados.

Préstamo de libro: Buscar por ISBN y cambiar prestado = true.

Listado de libros disponibles: Mostrar libros con prestado = false, ordenados por título.

Listado de libros prestados: Mostrar libros con prestado = true, ordenados por ISBN.

Devolver libro: Buscar por ISBN y cambiar prestado = false.

Borrar libro: Eliminar libro por ISBN.

Salir.

Ejercicio 2 - Hospital (Map Obligatorio)
Desarrollar una aplicación que gestione los pacientes de un hospital.

Requisitos obligatorios
Crear una clase llamada Paciente.
Utilizar como estructura principal un Map<String, Paciente>, donde:

La clave será el código del paciente.

El valor será el objeto Paciente.

Datos de cada paciente

codigo (String) → clave única

nombre (String)

edad (int)

habitacion (int)

dadoAlta (boolean)

Clase Paciente
Debe incluir:

Constructor/es necesarios.

Getters y setters.

Método toString() que muestre todos los datos.

Condiciones iniciales
Crear 5 pacientes en tiempo de diseño.
Todos estarán inicialmente con dadoAlta = false.

Opciones del menú

Alta de paciente: Registrar nuevo paciente. No se permite código duplicado.

Modificar habitación: Buscar paciente por código y actualizar la habitación.

Listado de pacientes en hospital: Mostrar pacientes con dadoAlta = false, ordenados por nombre.

Listado de pacientes dados de alta: Mostrar pacientes con dadoAlta = true.

Dar de alta: Buscar por código y cambiar dadoAlta = true.

Borrar paciente: Eliminar paciente por código.

Salir.
​

Criterios de evaluación (igual que el original):

Correcto control de duplicados.

Uso correcto de ArrayList y Map.

Organización clara del menú.

Métodos separados para cada funcionalidad.

Código limpio y funcional.
